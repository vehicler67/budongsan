## 1. 핵심 문제 정의

- 대상: 한국 등기부등본 PDF (고정 양식 + CID 폰트 섞임)
- 목표: 원본 구조를 최대한 보존한 채 텍스트·표·격자 레이아웃을 추출
- 현재 이슈: `parser.py`, `exporter.py` 빌드 테스트는 성공했으나
  - CID 폰트 PDF에서 텍스트 깨짐
  - OCR 결과 정제 부족
  - 토큰 사용량 급증으로 전면 재개 필요

## 2. 주요 오픈소스 도구

### 2.1 LiteParse (LlamaIndex)
- 공개: 2026-03-19 (LlamaIndex 오픈소스)
- 특징:
  - 로컬 실행, GPU 불필요, 무료
  - X, Y 좌표 공간 인식 기반 Grid Projection
  - 표/차트 레이아웃 보존에 특화
  - LLM을 뒤에 둔 "레이아웃 그대로 전달" 철학
- 참고:
  - `https://github.com/run-llama/liteparse`
  - `https://www.llamaindex.ai/blog/liteparse-local-document-parsing-for-ai-agents`

### 2.2 Open-Parse
- 특징:
  - 시각적 분석 기반 마크다운 추출
  - 헤더, 다중 컬럼, 중첩 테이블 보존
- 참고:
  - `https://github.com/...` (현재 조사 중)

### 2.3 Marker
- 특징:
  - GPU 활용 고속 파서
  - PDF 레이아웃·단락 자동 분할
- 참고:
  - `https://github.com/...` (현재 조사 중)

### 2.4 Docling (IBM)
- 특징:
  - 오픈소스, 자체 호스트 가능
  - 테이블 정확도 비교 벤치마크에서 상위권
- 참고:
  - `https://github.com/DS4SD/docling`

### 2.5 MinerU / OmniDocBench
- MinerU: 다중 PDF 파서 통합 평가 도구
- OmniDocBench: CVPR 2025 벤치마크 (1651 페이지, 10 document types)
- 참고:
  - `https://github.com/opendatalab/OmniDocBench`
  - `https://liduos.com/en/posts/ai-develope-tools-series-2-open-source-doucment-parsing/`

## 4. CID 폰트 텍스트 깨짐 관련

### 4.1 원인 요약
- 한국 등기부등본 PDF에 자주 등장하는 CID 폰트는 일반 텍스트 추출 라이브러리에서 글자 매핑이 깨짐
- `pdftotext`, `pymupdf` 등으로 단순 추출 시 원문 복원 불가

### 4.2 검색된 참고
- StackOverflow: "Extracting Text from a PDF with CID fonts"
  - `https://stackoverflow.com/questions/33413632/extracting-text-from-a-pdf-with-cid-fonts`
- Tesseract OCR 우회 전략:
  - PDF → 이미지 변환 후 Tesseract 실행
  - LLM으로 OCR 오류 후처리 (LLM-aided OCR)
- 관련 참고:
  - `https://github.com/tesseract-ocr/tesseract/issues/4466`
  - `https://news.hada.io/topic?id=16253`

  1) 기존 VB 플러그인의 구조·알고리즘 이해

  3) 한국어/CID 폰트 취약점은 이미지 기반 OCR + LLM 보정으로 우회
