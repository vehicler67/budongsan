## 3. 핵심 문제 해결

### 3.1 CID 폰트 문제 (가장 큰 난관)

| 단계 | 시도 | 결과 |
|------|------|------|
| 1 | PyMuPDF get_text() 직접 추출 | ❌ 각 글자 사이 공백 삽입 |
| 2 | pdfplumber 시도 | ❌ 동일 문제 |
| 3 | Tesseract OCR (400DPI) | ✅ 성공 |

**해결:** OCR로만 텍스트 추출 후 CID 공백을 정규식으로 제거.

```
"고 유 번 호" → fix_cid_spacing() → "고유번호"
"【 표 제 부 】"  → fix_cid_spacing() → "【표제부】"
```

**실패 사례:** `fix_cid_spacing()` 1회 실행 시 잔여 공백 발생 → 2회 실행으로 해결.

### 3.2 OCR 오류 정제

| 패턴 | 원인 | 해결 | 실패 사례 |
|------|------|------|-----------|
| "2023 년 2 월 20 일" | OCR 날짜 인식 | fix_date() | v6: 공백 미처리 |
| "제 107414 호" | OCR 등기번호 | fix_regno() | v6: "제 99956 호" 유지 |
| "금 266,000,000 원" | OCR 금액 인식 | fix_money() | v6: 공백 유지 |
| "【" → "[" | OCR 문자 오인식 | TERM_REPLACE | 【갑구】감지 실패 |
| "ㅣ" "｜" "∣" | OCR 유사 문자 | 정규식 | 노이즈 잔존 |

### 3.3 섹션 감지

v6에서는 CID 폰트 섹션 헤더(`【표제부】`)를 감지하지 못했다.
v7에서는 CID 공백 정제 후 TERM_REPLACE로 【→[ 변환까지 처리하여
4개 섹션 감지 성공.

**미감지:** `【갑구】`, `【을구】` — OCR이 【→ 다른 문자로 잘못 인식하는 경우
아직 일부 존재. TERM_REPLACE 지속 확장 필요.

### 3.4 표 구조 재현

PyMuPDF의 `get_drawings()`로 PDF 내 수평/수직선을 감지하여
표 영역(13개)을 추정했다. 현재는 영역 좌표와 행/열 수만 기록하고,
OCR 텍스트를 각 셀에 매핑하는 단계는 추가 구현이 필요하다.

## 4. 유지보수 가이드

### 4.1 새로운 PDF 처리시

```bash
python3 parser_v7.py
```

- 기본: `2849-2018-019318_25696174641_RIS.pdf` 처리
- 다른 PDF: `run()` 함수에 pdf_path 인자 전달
- 20페이지 이상 문서도 자동 처리

### 4.2 정확도 개선 방법

1. **TERM_REPLACE 확장**: OCR 오류 패턴 발견시 사전에 추가
2. **OCR DPI 조정**: `OCR_DPI = 400` → 600 (느리나 정확)
3. **추가 정규식**: KEYWORDS_CHECK에 누락 키워드 추가
4. **이미지 전처리 조정**: threshold 175 값을 조절

### 4.3 자주 발생하는 오류

| 오류 | 원인 | 확인 방법 |
|------|------|-----------|
| "고 유 번 호" 유지 | CID 정제 실패 | `experiments/v7_combined.txt` 확인 |
| 섹션 헤더 미감지 | OCR [→【 변환 실패 | combined.txt에서 "표제부" 검색 |
| 20페이지 이상 중단 | DPI 너무 높음 | OCR_DPI를 300으로 낮춤 |
| Excel 출력 실패 | openpyxl 오류 | `pip install openpyxl` 확인 |
| Tesseract 오류 | 언어팩 누락 | `tesseract --list-langs` → kor 확인 |

### 4.4 Docker/CI 환경 구성

```bash
# 필요 패키지
pip install pytesseract openpyxl PyMuPDF Pillow

# Tesseract 설치 (macOS)
brew install tesseract tesseract-lang

# 한글 언어팩
tesseract --list-langs | grep kor
```
